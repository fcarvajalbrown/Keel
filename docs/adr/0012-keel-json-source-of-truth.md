# ADR-0012: keel.json as the per-project source of truth

- Status: Accepted
- Date: 2026-06-14
- Deciders: Felipe Carvajal Brown

## Context

Per-song settings (which file is which label, per-label balance/pan/spread, the
master target) need a home that a non-programmer can edit and that is stable
across runs — not Python, not CLI flags retyped every time.

## Decision

Each song folder gets a **`keel.json`** mapping document, auto-generated by
`build.py` on first run (labels auto-detected) and then **hand-edited by the
user**. It is the **source of truth** per song: `{stems, balance, pan, spread,
glue, master}`. Re-running applies it. CLI flags (`--lufs/--tp/--ref/--preset/
--glue`) override it at render time but do not silently rewrite it.

## Consequences

- A song is reproducible from its folder alone; the mapping travels with the stems.
- Editing is JSON, not code; the auto-detect pass just seeds a starting point.
- CLI overrides are transient by design (see [ADR-0014](0014-named-master-presets.md)).
